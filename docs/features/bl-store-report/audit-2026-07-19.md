## 2. The decision set (ranked by observed use)

1. **STR** — qty basis, UK-first, world fallback accepted when labelled. First follow-up ~80% of the time.
2. **Honest net £** — full 9.4% fees + FULL standalone inbound postage on whatever subset is bought.
3. **Ask vs 6MA benchmark → margin %**, with provenance (uk / world† / none) — provenance challenged on 2 of 3 mornings, both challenges found real bugs.
4. **Overlap vs own stock** — NEW / R-OUT / R-THIN / DUP; DUPs excluded from the actionable figure.
5. **Magnets** — worldwide supply ≤3 lots + decent STR.
6. **Benchmark coverage split**.
7. **Demand cap — advisory, not filter** ("I can make a decision on the high volume ones case by case").
8. **Sold-price ceiling** — the sw0775 lesson.
9. **STR-gate ladder × overlap**.

Near-zero decision weight: the 0–100 grade, seller feedback stats, orders/mo.

## 3. The circles (inconsistencies found)

- STR basis flip-flopped (lots vs qty; UK vs worldwide) across surfaces.
- Fee stack declared in 4 places (once hardcoded as `0.094`); quoted as 9.4% / ~9% / 8.9% in different analyses.
- Postage: assess lens ex-postage, basket proportional, gate subsets under-charged until PR #618.
- Five renderers of the same data (assessment format.ts, discord-card, bl-basket renderReport, bl-pg-store-scan buildReport, React AssessmentView) — none sharing a formatter; chat improvised a sixth.
- Verdicts not comparable night-to-night across engine-bug fixes; Discord card led with the most flattering (uncapped) net.

## 4. What was built (this branch)

- **`src/lib/bricklink/fees.ts`** — canonical fee stack, STR gate ladder, liquid gate (0.25), magnet def, price bands. All four former declaration sites now import it.
- **Engine v7** — `marketSoldQty6mo` (demand-cap input) + `soldShareAtList` (ceiling signal, from the UK sold histogram) on every ScoredLot; `assembleAssessmentWithLots` exposes the full scored set.
- **`src/lib/bl-store-report`** — the common decision report: lens-neutral rows (assess + basket adapters), the honesty ladder **raw → demand-capped → LIQUID** (STR≥0.25, no DUPs, capped, standalone postage), the single canonical lot table, gate ladder × overlap, one formatter set, CLI + full-markdown renderers.
- **`scripts/store-report.ts`** — renders from persisted `bl_store_scrapes` offline (no Chrome, no API). The sanctioned path for conversational queries (`--magnets`, `--min-str`, `--no-dups`, `--pricing-lens`).
- **Wiring** — both lens CLIs emit the report automatically; `assessment.decision` stamps the ladder onto persisted rows; the Discord card now leads with LIQUID; bl-basket's PARTIAL DATA gap gate (PR #619) surfaces as an explicit caveat in the common report.
- **Mandate** — CLAUDE.md standard-pattern section + `.claude/commands/bl-basket.md`: every store-review answer renders through this module.

Validation on Blanco_Brix (the trigger store): £1,481 raw → £156 capped → **£47.78 liquid** (estimate lens; the Jul 19 £17.30 figure was the grounded lens — both one flag apart, same table).
